# 📊 QueryDaily GA4 시각화 가이드

> **목표:** GA를 처음 보는 팀원도 5분 만에 핵심 지표를 확인할 수 있는 대시보드 구성

## 🎯 우리가 수집하는 데이터 (한눈에 보기)

### 현재 추적 중인 핵심 이벤트 10개

| 이벤트 | 의미 | 활용 |
|--------|------|------|
| `page_view` | 페이지 방문 | 전체 트래픽 |
| `scroll_depth` | 스크롤 깊이 (25/50/75/100%) | 콘텐츠 참여도 |
| `view_item` | 상품 조회 (1초 이상) | 상품별 관심도 |
| `view_section` | 섹션 진입 | 콘텐츠 효과 |
| `click_cta` | 버튼/링크 클릭 | CTA 효과 측정 |
| `form_step_complete` | 폼 단계 완료 | 전환 퍼널 분석 |
| `form_field_interaction` | 필드 상호작용 | 폼 이탈 분석 |
| `page_engagement` | 페이지 이탈 시 | 체류시간/이탈 지점 |
| `sign_up` | 베타 가입 완료 | **핵심 전환** |
| `purchase` | 구매 완료 | **매출 전환** |

### 수집하는 세부 정보 (파라미터)

```
사용자 행동:
├─ 어떤 버튼 클릭? → cta_text, cta_location
├─ 어떤 상품 관심? → product_id, item_name, price
├─ 어디까지 스크롤? → percentage, max_scroll_depth
├─ 얼마나 머물렀나? → engagement_time_seconds
├─ 어디서 이탈? → exit_scroll_depth
└─ 폼 어디까지? → form_name, step_number
```

## 🚀 STEP 1: 필수 설정 (GA4 Console)

> ⚠️ 이 단계를 먼저 완료해야 시각화가 가능합니다!

### 1-1. 커스텀 차원 생성 (7개 필수)

**경로 (2025년 최신):**
```
GA4 홈 > 왼쪽 사이드바 하단 "관리" (⚙️ 톱니바퀴)
> 중간 열 "속성" 섹션 > 아래로 스크롤
> "데이터 표시" 섹션 > "맞춤 정의" 클릭
> 우측 상단 "맞춤 측정기준 만들기" 버튼
```

**생성할 차원:**

| 번호 | 측정기준 이름 | 범위 | 이벤트 매개변수 | 용도 |
|------|--------------|------|----------------|------|
| 1 | `cta_location` | 이벤트 | `cta_location` | CTA 위치별 효과 비교 |
| 2 | `cta_text` | 이벤트 | `cta_text` | 버튼 텍스트별 클릭률 |
| 3 | `product_id` | 이벤트 | `product_id` | 상품별 전환 분석 |
| 4 | `form_name` | 이벤트 | `form_name` | 폼 종류별 성과 |
| 5 | `section_name` | 이벤트 | `section_name` | 섹션별 효과 |
| 6 | `field_name` | 이벤트 | `field_name` | 이탈 필드 식별 |
| 7 | `interaction_type` | 이벤트 | `interaction_type` | 폼 상호작용 유형 |

**입력 예시 (1번):**
```
측정기준 이름: cta_location
범위: 이벤트
설명: CTA 버튼의 위치 (hero, product, navigation 등)
이벤트 매개변수: cta_location
```

**완료 확인:**
```
맞춤 정의 목록에 7개가 모두 보이면 성공 ✅
```

### 1-2. 커스텀 측정항목 생성 (6개 필수)

**같은 페이지에서:**
```
"맞춤 정의" 페이지 > 우측 상단 "맞춤 측정항목 만들기" 버튼
```

| 번호 | 측정항목 이름 | 측정 단위 | 이벤트 매개변수 | 용도 |
|------|--------------|----------|----------------|------|
| 1 | `engagement_time_seconds` | 초 | `engagement_time_seconds` | 평균 체류시간 |
| 2 | `exit_scroll_depth` | 표준 | `exit_scroll_depth` | 이탈 시 스크롤 % |
| 3 | `max_scroll_depth` | 표준 | `max_scroll_depth` | 최대 스크롤 % |
| 4 | `time_to_scroll` | 초 | `time_to_scroll` | 스크롤 도달 시간 |
| 5 | `step_number` | 표준 | `step_number` | 폼 단계 번호 |
| 6 | `view_duration` | 초 | `view_duration` | 상품 조회 시간 |

**입력 예시 (1번):**
```
측정항목 이름: engagement_time_seconds
측정 단위: 초
설명: 페이지 총 체류 시간
이벤트 매개변수: engagement_time_seconds
```

### 1-3. 전환 이벤트 마킹 (3개 필수)

**경로 (2025년 최신):**
```
옵션 A (추천):
GA4 홈 > 관리 > 속성 > "데이터 표시" 섹션 > "이벤트" 클릭

옵션 B:
GA4 홈 > 관리 > 속성 > 아래로 스크롤 > "전환" 클릭
```

**방법 1: 이벤트 목록에서 토글 ON (가장 빠름)**

```
"이벤트" 페이지에서 다음 이벤트를 찾아 우측 토글 스위치를 ON:

□ sign_up          → [토글 ON] ✅
□ purchase         → [토글 ON] ✅
□ begin_checkout   → [토글 ON] ✅
```

**방법 2: 수동 생성 (이벤트가 아직 발생 안 했을 경우)**

```
"전환" 페이지 > "새 전환 이벤트" 버튼 클릭
> 이벤트 이름 정확히 입력: sign_up
> 저장
```

**완료 확인:**
```
이벤트 목록에서 "전환" 열에 체크 표시 또는
전환 페이지에 3개 이벤트가 나타나면 성공 ✅
```

### 1-4. DebugView로 검증 (실시간 확인)

**경로:**
```
GA4 홈 > 왼쪽 사이드바 > "구성" > "DebugView"
```

**사전 준비:**
1. Chrome 확장 프로그램 설치: [Google Analytics Debugger](https://chrome.google.com/webstore/detail/google-analytics-debugger/jnkmfdileelhofjcijamephohjechhna)
2. 확장 프로그램 활성화 (아이콘 클릭 → 파란색)
3. QueryDaily 사이트 새로고침

**확인 체크리스트:**

```
사이트 접속 → DebugView에서 확인:

✅ page_view 이벤트 발생
✅ scroll_depth (25, 50, 75, 100 순차 발생)
✅ view_item (상품 카드 보면 발생, product_id 확인)
✅ click_cta (버튼 클릭 시, cta_location + cta_text 확인)
✅ view_section (섹션 진입 시, section_name 확인)
✅ form_step_complete (폼 진행 시, step_number 확인)
✅ page_engagement (탭 닫을 때, engagement_time_seconds 확인)
```

**이벤트 상세 보기:**
```
DebugView에서 이벤트 클릭 → 우측 패널에서 매개변수 확인

예시: click_cta 클릭 시
├─ cta_text: "지금 시작하기 - 그로스 플랜"
├─ cta_location: "product"
├─ product_id: "growth-plan"
└─ scroll_depth_at_click: 45
```

## 📊 STEP 2: 핵심 리포트 5개 만들기

> 💡 **팁:** 각 리포트는 5분 안에 생성 가능합니다. 한 번 만들면 자동 업데이트!

### 리포트 1: CTA 클릭 효과 분석 (난이도: ⭐)

**목적:** 어떤 버튼이 가장 많이 클릭되는가?

**생성 방법:**
```
GA4 홈 > 왼쪽 사이드바 > "탐색" (🔍)
> 우측 상단 "+" 버튼 또는 "새로운 탐색 만들기"
> 템플릿: "자유 형식" 선택
```

**설정:**

| 항목 | 설정값 |
|------|--------|
| **탐색 이름** | "CTA 클릭 효과" |
| **날짜 범위** | 지난 28일 |
| **차원** | 1. 이벤트 이름<br>2. cta_location (맞춤 차원)<br>3. cta_text (맞춤 차원)<br>4. product_id (맞춤 차원) |
| **측정항목** | 1. 이벤트 수<br>2. 총 사용자 |
| **필터** | 이벤트 이름 = `click_cta` |
| **행** | cta_location, cta_text |
| **값** | 이벤트 수, 총 사용자 |

**시각화 옵션:**
```
우측 상단 차트 유형:
- 표 (기본) ← 추천
- 막대형 차트
- 파이 차트
```

**완성 화면 예시:**
```
┌──────────────────────────────────────────────┐
│ cta_location  │ cta_text           │ 이벤트 수 │
├──────────────────────────────────────────────┤
│ product       │ 지금 시작하기 - 그로스 플랜  │   45    │
│ navigation    │ 왜 QueryDaily      │   32    │
│ hero          │ 상품 선택하기       │   28    │
│ faq_accordion │ FAQ 1             │   15    │
└──────────────────────────────────────────────┘
```

**인사이트 읽는 법:**
- ✅ **가장 위**: 가장 많이 클릭되는 CTA
- ✅ **product_id 필터**: 상품별 성과 비교
- ✅ **날짜 범위 변경**: 주간/월간 트렌드 확인

### 리포트 2: 전환 퍼널 분석 (난이도: ⭐⭐)

**목적:** 사용자가 어디서 이탈하는가?

**생성 방법:**
```
GA4 홈 > 탐색 > "새로운 탐색 만들기"
> 템플릿: "유입경로 탐색 분석" 선택
```

**설정:**

| 항목 | 설정값 |
|------|--------|
| **탐색 이름** | "베타 신청 퍼널" |
| **날짜 범위** | 지난 28일 |
| **유입경로 유형** | 열린 유입경로 (중간 건너뛰기 허용) |

**단계 구성 (7단계):**

| 단계 | 이벤트 이름 | 조건 | 의미 |
|------|------------|------|------|
| 1 | `page_view` | page_path = `/` | 랜딩페이지 방문 |
| 2 | `scroll_depth` | percentage ≥ 50 | 절반 이상 스크롤 |
| 3 | `view_item` | (조건 없음) | 상품 조회 |
| 4 | `click_cta` | (조건 없음) | CTA 클릭 |
| 5 | `form_field_interaction` | interaction_type = `focus` | 폼 시작 |
| 6 | `form_step_complete` | step_number ≥ 2 | 2단계 이상 진행 |
| 7 | `sign_up` | (조건 없음) | 가입 완료 |

**단계 추가 방법:**
```
좌측 패널 "탭 설정" > "단계" 섹션

각 단계:
1. [단계 추가] 버튼 클릭
2. 단계 이름 입력 (예: "방문")
3. 이벤트 선택 또는 직접 입력
4. 조건 추가 (필요 시): [조건 추가] 클릭
```

**완성 화면 예시:**
```
┌──────────────────────────────────────────┐
│ 방문           1,000명 (100%)            │
│     ↓ 이탈 200명 (20%)                   │
│ 절반 스크롤       800명 (80%)            │
│     ↓ 이탈 200명 (25%)                   │
│ 상품 조회         600명 (60%)            │
│     ↓ 이탈 150명 (25%)                   │
│ CTA 클릭         450명 (45%)            │
│     ↓ 이탈 150명 (33%)                   │
│ 폼 시작           300명 (30%)            │
│     ↓ 이탈 120명 (40%)  ⚠️ 병목 구간!   │
│ 2단계 진행        180명 (18%)            │
│     ↓ 이탈 80명 (44%)   ⚠️ 병목 구간!   │
│ 가입 완료         100명 (10%)            │
└──────────────────────────────────────────┘
```

**인사이트 읽는 법:**
- ⚠️ **이탈률 40% 이상**: 즉시 개선 필요
- ✅ **단계별 비교**: 어디가 병목인지 한눈에 파악
- 📈 **시간 필터**: 개선 전후 비교

### 리포트 3: 폼 이탈 분석 (난이도: ⭐)

**목적:** 어떤 단계에서 폼을 이탈하는가?

**생성 방법:**
```
GA4 홈 > 탐색 > 자유 형식
```

**설정:**

| 항목 | 설정값 |
|------|--------|
| **탐색 이름** | "폼 단계별 이탈" |
| **차원** | 1. form_name (맞춤 차원)<br>2. step_number (맞춤 측정항목) |
| **측정항목** | 1. 이벤트 수<br>2. 총 사용자 |
| **필터** | 이벤트 이름 = `form_step_complete` |
| **행** | form_name, step_number |
| **값** | 이벤트 수 |

**시각화:**
```
차트 유형: 막대형 차트 (가로)
X축: step_number
Y축: 이벤트 수
```

**완성 화면 예시:**
```
┌─────────────────────────────────────┐
│ beta_signup                         │
├─────────────────────────────────────┤
│ 1단계 ████████████████████ 200     │
│ 2단계 ██████████████ 140            │
│ 3단계 ████████ 80                   │
│ 4단계 ████ 40                       │
└─────────────────────────────────────┘

각 단계 완료율:
- 1→2: 70% (30% 이탈)
- 2→3: 57% (43% 이탈) ⚠️ 개선 필요!
- 3→4: 50% (50% 이탈) ⚠️ 개선 필요!
```

**인사이트 읽는 법:**
- 📉 **급격한 하락**: 해당 단계 UI/UX 문제
- ✅ **form_name 비교**: Beta vs Purchase 성과 비교
- 🎯 **개선 우선순위**: 이탈률 높은 단계부터

### 리포트 4: 페이지 참여도 분석 (난이도: ⭐)

**목적:** 사용자가 얼마나 머물고, 어디까지 스크롤하는가?

**생성 방법:**
```
GA4 홈 > 탐색 > 자유 형식
```

**설정:**

| 항목 | 설정값 |
|------|--------|
| **탐색 이름** | "페이지 참여도" |
| **차원** | 1. page_path |
| **측정항목** | 1. engagement_time_seconds (맞춤 측정항목)<br>2. exit_scroll_depth (맞춤 측정항목)<br>3. max_scroll_depth (맞춤 측정항목)<br>4. 총 사용자 |
| **필터** | 이벤트 이름 = `page_engagement` |
| **행** | page_path |
| **값** | 평균 engagement_time_seconds, 평균 max_scroll_depth |

**집계 설정:**
```
측정항목 클릭 > "평균" 선택 (기본값은 "합계")
```

**완성 화면 예시:**
```
┌────────────────────────────────────────────┐
│ page_path  │ 평균 체류(초) │ 평균 최대 스크롤 │
├────────────────────────────────────────────┤
│ /          │     127초     │      78%       │
│ /payment   │      65초     │      45%  ⚠️  │
│ /complete  │      15초     │      90%       │
└────────────────────────────────────────────┘
```

**인사이트 읽는 법:**
- ✅ **체류시간 60초 이상**: 관심 높음
- ⚠️ **스크롤 50% 미만**: 콘텐츠 개선 필요
- 📊 **exit_scroll_depth vs max**: 이탈 지점 파악

## 🎯 STEP 5: 실전 활용 시나리오

### 시나리오 1: 전환율이 갑자기 떨어졌다! 📉

**문제:**
```
월요일: 전환율 5.2%
화요일: 전환율 3.1% ⚠️ 40% 감소!
```

**원인 파악 (5분):**

```
1. 전환 퍼널 리포트 열기
   → 어느 단계에서 이탈 증가?

   결과: "폼 시작 → 2단계 진행" 이탈률
         30% → 60% 증가 발견!

2. 폼 이탈 분석 리포트 열기
   → step_number = 2 필터
   → field_name 확인

   결과: "email" 필드에서 에러 급증

3. DebugView로 실시간 확인
   → form_field_interaction 이벤트
   → interaction_type = "error" 필터
   → error_message 확인

   결과: "Email validation failed" 반복 발생
```

**해결:**
```
원인: 이메일 검증 로직 버그
조치: 즉시 핫픽스 배포
확인: 다음날 전환율 5.1% 회복 ✅
```

   ┌─────────────────────────────┐

   ├─────────────────────────────┤
