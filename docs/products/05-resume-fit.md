# 레주메 핏 (Resume Fit)

## 제품 정보
- **가격**: ₩59,000 (정가 ₩79,000)
- **형태**: 이력서 전문가 분석
- **내용**: 전문가 분석 + 맞춤 개선 가이드 + 예상 질문 15개

---

## 철학

### "좋은 이력서는 면접 질문의 80%를 결정합니다."

면접에서 나오는 질문의 80%는
면접관이 당신의 이력서를 보고 떠올린 질문입니다.

"이 사람이 Redis를 도입했다고? → Redis 질문 준비"
"JPA 최적화를 했다고? → N+1 질문 준비"
"트래픽 개선했다고? → 얼마나 개선됐는지 물어봐야지"

이력서가 명확하면 → 예상 가능한 질문
이력서가 애매하면 → 예상 못한 질문

레주메 핏은
당신의 이력서를 '면접관의 눈'으로 분석합니다.

어떤 질문이 나올지 예측하고,
그 질문에 답할 수 있도록 이력서를 개선합니다.

이력서를 고치는 게 아니라,
면접을 준비하기 쉽게 만드는 겁니다.

에너지는 낮게 (☕☕)
준비는 철저하게.

## 작동 방식

### Step 1: 이력서 제출 (5분)

```
📝 이력서 업로드 (PDF, DOC, TXT)
   → 지원 분야 선택
      (백엔드, 프론트엔드, 풀스택, DevOps 등)
   → 목표 회사 입력 (선택)
      예: "네이버, 카카오 지망"
```

---

### Step 2: 전문가 분석 (24시간 이내)

```
👨‍💼 현직 시니어 개발자가 분석 시작

🔍 분석 관점 5가지

1. 면접관의 눈으로 읽기
   "이 이력서를 보면 무슨 질문이 떠오르는가?"

2. 약점 찾기
   "어떤 부분에서 막힐 것 같은가?"

3. 강점 찾기
   "어떤 부분을 더 부각시킬 수 있는가?"

4. 수치 확인
   "구체적 수치가 있는가? 없으면 어디에 넣어야 하는가?"

5. 기술 깊이 평가
   "기술을 얼마나 깊게 이해하고 있는가?"
```

## 종합 분석 리포트

### 📊 종합 평가

```
완성도: 7/10
명확성: 6/10 ⚠️ 개선 필요
기술 깊이: 8/10
수치 활용: 5/10 ⚠️ 개선 필요
```

---

### ✅ 강점 3가지

#### 1. 프로젝트 다양성

```
Spring, Redis, AWS 등 다양한 기술 스택 경험
→ 면접관의 흥미 유발 가능

💡 활용 팁
"이 중 하나를 '메인 무기'로 선택하세요.
 예: 'Spring JPA 최적화 전문가'"
```

#### 2. 트러블슈팅 경험

```
"프로덕션 장애 해결" 경험 명시
→ 실전 능력 증명

💡 활용 팁
"장애 해결 프로세스를 STAR 구조로 정리하세요.
 면접에서 90% 물어봅니다."
```

#### 3. 팀 협업 경험

```
"5명 팀 리드" 경험
→ 리더십 잠재력

💡 활용 팁
"팀원과의 갈등 해결 사례를 준비하세요.
 팀 리드 경험자는 필수 질문입니다."
```

---

### ⚠️ 약점 3가지 (개선 필요)

#### 1. 수치 부족 ⚠️

**[현재 이력서]**

```
"API 응답 속도를 개선했습니다."
```

**❌ 문제점**

```
→ 얼마나? 어떻게? 왜?
→ 면접관이 꼬리 질문으로 파고들 지점
```

**[개선 제안]**

```
"API 응답 속도를 800ms→120ms로 85% 개선했습니다.
 (Fetch Join 적용, 쿼리 수 50→2개 감소)"
```

**✅ 개선 효과**

```
→ 구체적 수치 = 신뢰도 ↑
→ 꼬리 질문 대비 가능
```

---

#### 2. 기술 선택 이유 없음 ⚠️

**[현재 이력서]**

```
"Redis를 도입했습니다."
```

**❌ 문제점**

```
→ "왜 Redis? MongoDB는?" 질문 예상
→ 대답 못 하면 "그냥 따라 했구나" 인상
```

**[개선 제안]**

```
"세션 데이터 캐싱을 위해 Redis를 선택했습니다.
 (TTL 관리 편의성, 인메모리 속도)"
```

**✅ 개선 효과**

```
→ 의사결정 능력 증명
→ "왜?" 질문 대비 완료
```

---

#### 3. 애매한 표현 ⚠️

**[현재 이력서]**

```
"프로젝트에 기여했습니다."
```

**❌ 문제점**

```
→ "뭘 했는데?" (면접관 혼란)
→ 막연함 = 신뢰도 ↓
```

**[개선 제안]**

```
"프로젝트의 인증 모듈을 설계하고 구현했습니다.
 (Spring Security + JWT, 동시 접속 1,000명 처리)"
```

**✅ 개선 효과**

```
→ 역할 명확
→ 질문 포인트 예측 가능
```

### 🎯 예상 면접 질문 15개

당신의 이력서를 보고 면접관이 물어볼 질문 예측

#### 📋 기술 경험 (70% 확률)

```
1. "Redis를 왜 선택했나요?" (필수)
2. "TTL은 어떻게 설정했나요?" (꼬리)
3. "N+1 문제 해결 과정은?" (필수)
4. "Fetch Join의 단점은?" (꼬리)
5. "API 응답 속도 개선 수치는?" (필수)
```

#### 📋 트러블슈팅 (50% 확률)

```
6. "프로덕션 장애 해결 사례는?" (필수)
7. "원인 파악은 어떻게 했나요?" (꼬리)
8. "재발 방지책은?" (꼬리)
```

#### 📋 팀 협업 (30% 확률)

```
9. "팀원과 갈등 경험은?"
10. "코드 리뷰는 어떻게 진행했나요?"
```

#### 📋 기타 (보너스)

```
11. "가장 어려웠던 기술적 결정은?"
12. "다음 프로젝트에서 개선할 점은?"
13. "Spring을 왜 선택했나요?"
14. "테스트 커버리지는?"
15. "Git 전략은?"
```

**💡 준비 우선순위**

```
1-5번 질문: 90% 나옴, 필수 준비
6-10번 질문: 50% 나옴, 준비 권장
11-15번 질문: 보너스, 시간 있으면
```

### 📝 개선된 이력서 (예시)

**[Before]**

```
"API 응답 속도를 개선했습니다."
```

**[After]**

```
"API 응답 속도를 800ms→120ms로 85% 개선 (Fetch Join 적용)
 - 문제: N+1 쿼리로 인한 응답 지연 (평균 800ms)
 - 해결: Fetch Join 적용, 쿼리 수 50개→2개 감소
 - 결과: 응답 속도 85% 개선 (120ms), 동시 처리량 2배 증가"
```

**💡 개선 포인트**

```
✅ 수치 3개 추가 (Before, After, 개선율)
✅ 기술 명시 (Fetch Join)
✅ 문제→해결→결과 구조
```
