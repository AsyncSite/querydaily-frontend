# v1: "따듯한 선배" (The Warm Mentor)

## 🎯 핵심 컨셉
**"당신은 이미 충분한 개발자입니다. 단지 보여주는 방법을 몰랐을 뿐."**

## 💡 전략 철학

### 브랜드 포지셔닝
같은 길을 걸었던 선배 개발자가, 카페에서 후배에게 조언하듯
압박 없이, 격려하며, 함께 가는 느낌

### 감정의 흐름
```
막막함, 외로움
    ↓ (공감)
"나만 그런 게 아니구나"
    ↓ (신뢰)
"이 사람들은 날 이해해"
    ↓ (안내)
"이렇게 하면 되는구나"
    ↓ (행동)
"시작해볼까?"
```

## 🎨 톤앤매너

### 금지어
- ❌ 커피값, 저렴함, 할인, 할인율
- ❌ 긴급, 마감임박, 한정수량, 지금 바로
- ❌ "답 못하시죠?", "떨어지는 이유"
- ❌ 100% 합격, 무조건, 반드시

### 권장어
- ✅ 함께, 충분히, 이미, 단지
- ✅ 천천히, 괜찮아요, 그 마음 압니다
- ✅ 준비 중, 성장, 변화, 경험

### 문체 원칙
1. **공감 먼저**: "그 마음, 압니다"로 시작
2. **격려 중심**: "당신은 이미 충분합니다"
3. **구체적 안내**: "이렇게 해보세요"
4. **동행 약속**: "함께 가겠습니다"

## 📋 메시지 전략

### Hero 섹션 - 극도의 미니멀리즘
```
[Before - 10개 요소]
- Badge
- Title (2줄)
- Subtitle (2줄)
- Description (3줄)
- CTA 버튼 2개
- Trust Badge 2개
= 사용자 압도, 집중 분산

[After - 3개 요소]
Title (2줄):
"혼자 준비하고 계신가요?
함께 가겠습니다."

Subtitle (1줄, 작게):
"100명이 넘는 개발자가 함께 준비하고 있습니다"

CTA (1개):
"시작해보기"

= 명확한 메시지, 단일 행동
```

### 섹션 구성 순서

#### 1. Hero - 공감 + 약속
**목적**: 첫 3초 안에 공감 형성
**메시지**: "혼자가 아니에요, 함께 가요"
**요소**: Title, Subtitle, CTA (3개만)

#### 2. 문제 공감 - "당신만 그런 게 아니다"
**목적**: 채용 시장 현실 인식
**메시지**: "내가 부족한 게 아니에요"

**통계 3개** (큰 숫자로 임팩트):
```
87%
"경력 1년 이상" 요구 공고 비율
(당신 탓이 아닙니다)

142:1
대기업 개발자 평균 경쟁률
(누구나 어렵습니다)

3.2초
면접관이 첫인상 판단하는 시간
(준비가 필요합니다)
```

**공감 메시지**:
```
"내가 부족해서가 아닙니다.
시장이 변했고, 면접 방식이 변했습니다.
당신은 충분히 좋은 개발자예요."
```

#### 3. 우리의 이야기 - 신뢰 구축
**목적**: "이 사람들 나를 이해해"
**스토리**: 실패 → 깨달음 → 변화 → 도움

```
타임라인 형태:

2019년
└─ "저도 100번 넘게 떨어졌습니다"
   기술은 알았지만, 설명을 못했어요

2020년
└─ "질문 패턴을 분석하기 시작했습니다"
   면접관이 뭘 듣고 싶어하는지 공부했어요

2021년
└─ "매일 질문을 만들고 답해봤습니다"
   혼자 연습하며 자신감이 생겼어요

2022년
└─ "이제는 면접관으로 일합니다"
   같은 고민을 하는 후배들을 봅니다

2024년
└─ "제가 겪은 시행착오, 당신은 겪지 마세요"
   함께 준비하겠습니다
```

#### 4. 해결 방법 - 명확한 3단계
**목적**: "이렇게 하면 되는구나"
**메시지**: 복잡하지 않아요, 이것만 하세요

```
Step 1. 이력서 업로드
"당신의 경험을 이해합니다"
→ 5분이면 충분해요

Step 2. 매일 질문 받기
"실제로 나올 질문을 예측합니다"
→ 출근길 10분만 읽어보세요

Step 3. 스스로 답 찾기
"고민하는 과정이 실력이 됩니다"
→ 답을 외우는 게 아니에요
```

#### 5. 실제 변화 - Before & After
**목적**: 구체적 증거
**메시지**: "이렇게 바뀝니다"

```
3개의 변화 스토리:

Story 1: 기술 설명
Before: "JPA 왜 썼어요?"
        → "그냥... 팀에서 쓰니까..."
After:  "캐싱 전략과 N+1 문제 해결을 고려해서
        JPA를 선택했고, 쿼리 최적화를 통해..."

Story 2: 자신감
Before: "긴장하면 머릿속이 하얘져요"
        → 예상 못한 질문에 당황
After:  "예상 질문 연습으로 자신감 생겼어요"
        → 침착하게 생각하고 답변

Story 3: 설득력
Before: "포트폴리오는 있는데 설명을 못해요"
        → 기능 나열만
After:  "기술 선택 이유를 논리적으로 설명해요"
        → 왜 이렇게 했는지 명확히
```

#### 6. 실제 후기
**목적**: 사회적 증거
**기존 유지**: 만족도 차트, 텍스트 리뷰

#### 7. 상품 - "이제 시작하는 방법"
**목적**: 자연스러운 전환
**메시지**: 가격이 아닌 가치 중심

**전환 메시지**:
```
"혼자 준비하는 건 외롭습니다.
함께 시작하는 방법입니다."
```

**상품 제시 방식**:
```
1. 크리티컬 히트
   "가볍게 시작해보기"
   → 핵심 질문 3개로 감 잡기
   → 부담 없는 첫 경험

2. 그로스 플랜 [MOST POPULAR]
   "본격적으로 준비하기"
   → 20일간 매일 질문
   → 체계적인 준비

3. 리얼 인터뷰
   "실전처럼 연습하기"
   → 90분 모의면접
   → 완벽한 마무리
```

**가격 표시 원칙**:
- ❌ "~~₩106,000~~ ₩49,000 (54% 할인)"
- ✅ "첫 도전 응원가 ₩49,000"
- ❌ "커피 2잔 가격"
- ✅ "하루 ₩2,450, 20일 투자"

#### 8. FAQ
**기존 유지**: 환불 정책, 품질 보장

## 🎯 타겟 고객

### 주요 타겟
- 신입/주니어 개발자 (0-3년차)
- 혼자 준비하다 막막함을 느끼는 사람
- 기술은 아는데 설명이 어려운 사람
- 누군가의 조언이 필요한 사람

### 페르소나: "혼자 준비하는 김개발"
```
이름: 김개발 (26세, 신입 개발자)
상황: 부트캠프 수료 → 100군데 지원 → 면접 3번 → 모두 탈락
고민:
  "포트폴리오는 있는데 면접에서 뭘 물어볼지 모르겠어요.
   ChatGPT에 물어봐도 일반적인 답변만 나오고...
   주변에 물어볼 선배도 없고, 혼자 준비하니까
   이게 맞는지 틀린지도 모르겠어요."

니즈:
  - 명확한 가이드
  - 실제 질문 예측
  - 혼자가 아닌 느낌
  - 따듯한 격려

QueryDaily의 역할:
  "제가 옆에서 함께 준비해드릴게요.
   당신은 이미 충분한 개발자예요."
```

## 💰 전환 시나리오

### 1단계: 공감 (Hero)
```
"혼자 준비하고 계신가요?"
→ "맞아, 나 혼자 하고 있어..."
→ "함께 가겠습니다"
→ "같이 하는 사람이 있구나"
```

### 2단계: 안도 (문제 공감)
```
통계 87%, 142:1 확인
→ "나만 어려운 게 아니구나"
→ "내가 부족한 게 아니었어"
→ 심리적 부담 감소
```

### 3단계: 신뢰 (우리의 이야기)
```
제작자 실패 스토리
→ "이 사람들도 그랬구나"
→ "진짜 나를 이해하는구나"
→ 신뢰 형성
```

### 4단계: 희망 (해결 방법)
```
3단계 확인
→ "이렇게 하면 되는구나"
→ "나도 할 수 있을 것 같아"
→ 행동 가능성 인식
```

### 5단계: 확신 (Before & After)
```
변화 스토리 확인
→ "진짜 바뀌는구나"
→ "나도 이렇게 바뀔 수 있어"
→ 전환 직전
```

### 6단계: 행동 (상품)
```
"이제 시작하는 방법입니다"
→ 자연스러운 전환
→ "시작해볼까?"
→ CTA 클릭
```

## 📊 기대 효과

### 정량적 목표
- Hero 이탈률: 50% → 30% (20%p 감소)
- 상품 섹션 도달률: 40% → 60% (20%p 증가)
- CTA 클릭률: 3% → 5% (67% 증가)
- 전환율: 1.5% → 2.5% (67% 증가)

### 정성적 목표
**사용자 피드백 기대:**
- "압박감 없이 편안했어요"
- "진짜 나를 이해해주는 것 같아요"
- "따듯한 느낌이 들어서 신뢰가 가요"
- "혼자가 아니라는 생각이 들었어요"

### 브랜드 이미지
**Before**:
- "저렴한 면접 준비 서비스"
- "할인하는 SaaS"

**After**:
- "함께하는 멘토"
- "따듯한 선배"
- "신뢰할 수 있는 파트너"

## 🚫 절대 하지 말아야 할 것

1. **가격으로 승부하지 않기**
   - 커피값, 저렴함 언급 금지
   - 할인율 강조 금지

2. **불안 자극하지 않기**
   - "떨어지는 이유", "실패" 같은 부정적 단어
   - "답 못하시죠?" 같은 도발적 질문

3. **압박하지 않기**
   - "지금 바로", "마지막 기회"
   - 타이머, 한정수량

4. **과장하지 않기**
   - "100% 합격", "무조건 성공"
   - "면접 끝판왕"

## ✅ 핵심 원칙

**"Less is More"**
- 적게 보여주되, 깊이 공감하기
- 많은 정보보다, 하나의 진심
- 화려한 디자인보다, 따듯한 메시지

**"Show, Don't Tell"**
- "좋은 서비스"라고 말하지 말고
- 실제 변화를 보여주기

**"People First"**
- 기능이 아닌 사람
- 가격이 아닌 가치
- 판매가 아닌 도움
