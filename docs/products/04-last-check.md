'최소한 이것만은 챙겼다'는 안정감입니다.

면접 D-1, 단 1시간으로
가장 자주 나오는 핵심 질문 15개를 완벽하게 정리합니다.

깊이보다는 넓이,
완벽보다는 안정감.

"이 15개만 답할 수 있으면, 최소한 떨어지진 않는다."

이 확신이 면접 전날의 패닉을 자신감으로 바꿉니다.

## 질문 구성

### 📋 Category 1: 자기소개 & 동기 (15분)

#### Q1. "간단히 자기소개 부탁드립니다."

**📝 템플릿 답변**

```
"안녕하세요, [이름]입니다.
 [경력]년 동안 [분야]에서 [핵심 경험]을 쌓았습니다.
 특히 [프로젝트 A]에서 [성과]를 달성했고,
 이 경험을 [지원 회사]에서 [어떻게 활용]하고 싶습니다."
```

**💡 체크포인트**

```
□ 1분 이내로 끝내기
□ 지원 회사와 연결하기
□ 구체적 성과 1개 넣기
```

#### Q2. "왜 우리 회사에 지원했나요?"

**📝 템플릿 답변 (빈칸 채우기 형식)**

```
"[회사의 특징]에 끌렸습니다.
 제 경험인 [프로젝트 A]가
 귀사의 [사업/제품]과 연결된다고 생각했고,
 특히 [기술/문화]에서 성장하고 싶습니다."
```

**💡 체크포인트**

```
□ 회사 홈페이지 핵심 키워드 1개
□ 내 경험과 연결
□ "배우고 싶다"보다 "기여하고 싶다"
```

#### Q3. "당신의 강점은?"

**🗣️ 만능 답변 (막힐 때 사용)**

```
"문제를 끝까지 파고드는 점입니다.
 [프로젝트 A]에서 [문제 상황],
 원인을 찾기 위해 [노력],
 결국 [해결] 했습니다."
```

**💡 체크포인트**

```
□ 강점 = 스토리로 증명
□ 결과 수치 포함
```


#### Q6. "JPA N+1 문제 경험은?"

**📝 3단계 답변**

```
1. 문제 발견: "쿼리가 50개씩 나가더라고요."
2. 해결 방법: "Fetch Join 적용했습니다."
3. 결과: "쿼리 2개로 줄었고, 응답 속도 800ms→120ms"
```

#### Q11. "가장 어려웠던 문제는?"

**📝 STAR 구조**

```
S (상황): "프로덕션에서 응답 속도가 5초 걸렸습니다."
T (과제): "원인을 찾아 개선해야 했습니다."
A (행동): "모니터링 확인 → DB 쿼리 분석 → 인덱스 추가"
R (결과): "5초→500ms로 개선"
```

```

 각각의 장단점을 정리해 비교하고,
 팀 상황에 맞는 [결정]을 함께 내렸습니다."
```
