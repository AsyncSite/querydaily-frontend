# v2: "면접관이 물을 질문, 미리 알고 가세요"

## 🎯 핵심 컨셉

**"예측 가능한 면접, 준비된 당신"**

면접 전날 밤, 당신은 검색합니다.
"Spring 면접 질문", "백엔드 면접 준비"

하지만 검색 결과는 너무 일반적입니다.
당신의 이력서에는 없는 기술, 당신이 하지 않은 프로젝트...

**면접관은 당신의 이력서를 보고 묻습니다.**
구글도 ChatGPT도 답을 모릅니다.

QueryDaily는 다릅니다.
- 당신의 이력서를 분석합니다
- 당신이 한 경험을 예측합니다
- 당신만의 답을 준비시킵니다

---

## 💡 컨셉의 핵심

### 고객이 진짜 원하는 것
```
❌ "면접 질문 리스트"  (정보)
✅ "면접장에서 당당하게 답하기"  (자신감)
```

### 감정의 여정
```
불안: "뭐 물어볼지 모르겠어요"
  ↓
예측: "이력서를 분석했습니다"
  ↓
준비: "매일 질문과 답변을 연습합니다"
  ↓
자신감: "이미 준비했어요"
```

### 차별점의 본질
```
일반 면접 준비:
구글 검색 → 일반론 암기 → 막연한 불안

QueryDaily:
이력서 분석 → 맞춤 예측 → 구체적 준비 → 당당한 면접
```

### 3. Solution - QueryDaily만의 해결책 (차별점)
**목적**: "왜 QueryDaily?"에 대한 명확한 답변

```
제목: "QueryDaily는 다릅니다"

비교:
일반 면접 준비 vs QueryDaily

차별점 3가지:

1️⃣ 이력서 기반 예측
   "당신의 경험, 당신의 기술로 질문을 만듭니다"

2️⃣ STAR 기법 가이드
   "경험을 구조화해서 답변하는 방법을 알려드립니다"

3️⃣ 매일 자동 발송
   "하루 10분, 매일 연습하면 자연스럽게 준비됩니다"
```

**실제 예시 (토글 인터랙션)**:
```
[일반 질문] vs [이력서 기반 질문]

일반:
"Spring Boot의 장점은?"
→ 외우기

이력서 기반:
"3번 프로젝트에서 Spring Boot를 선택한 이유는?
 Django가 아닌 이유는?"
→ 내 경험 말하기
```

## 🎨 디자인 원칙

### 1. Less is More
- 불필요한 애니메이션 제거
  - ❌ 3D 카드 플립, 시계 애니메이션
  - ✅ 페이드인, 슬라이드인만
- 섹션 간 여백 충분히
- 각 섹션의 목적 명확히

### 2. 타이포그래피 중심
```
H1: 4-5rem, font-weight 900 (Hero, 섹션 제목)
H2: 2-3rem, font-weight 700 (섹션 내 소제목)
Body: 1.2rem, font-weight 400 (본문)
Small: 1rem, opacity 0.7 (부가 정보)
```

### 3. 색상 전략
```
Primary: #6366F1 (Indigo) - CTA, 강조
Secondary: #8B5CF6 (Purple) - 배지, 액센트
Background: #F9FAFB (Gray-50) - 배경
Text: #111827 (Gray-900) - 본문
Light: #6B7280 (Gray-500) - 부가 정보
```

### 4. 레이아웃 원칙
- 중앙 정렬 (max-width: 1200px)
- 섹션 간 패딩: 8-10rem
- 카드 간 간격: 2rem
- 여백을 통한 시각적 호흡

### 5. 애니메이션 원칙
- 스크롤 트리거: Intersection Observer
- 페이드인: opacity 0 → 1 (0.6s ease)
- 슬라이드업: translateY(20px) → 0 (0.6s ease)
- hover: scale(1.02), shadow 증가
- transition: 0.3s ease
